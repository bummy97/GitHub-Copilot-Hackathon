## 1. 개요 (Overview)

사내 임직원이 업무용 AI 에이전트(Copilot Agent)를 **등록·검색·실행**할 수 있는
**간단한 웹 포털**을 구축한다. 포털은 **단순한 HTML 페이지**로 구현하며, 각 에이전트는
**URL 형태로 등록**되어 포털 안에서 **iframe으로 바로 열린다**. 별도 설치 없이
**브라우저만으로** 사용하는 것을 목표로 하며, 복잡한 권한·접근 제어 없이 누구나 쉽게
쓰는 데 초점을 둔다.

---

## 2. 배경 및 문제 정의 (Background & Problem)

- 부서마다 개별적으로 LLM 기반 도구를 만들어 **중복 개발**과 **품질 편차**가 발생한다.
- 좋은 에이전트가 만들어져도 **공유·발견할 수단이 없어** 재사용되지 못한다.
- 에이전트를 쓰려면 설정이 복잡해 **진입 장벽이 높다**.
- 이미 각 팀이 **자체 URL로 배포한 에이전트**가 흩어져 있어, 이를 **한곳에 모아 링크/임베드**할 통로가 없다.

---

## 3. 목표 및 비목표 (Goals / Non-Goals)

### 3.1 목표
- 에이전트 **카탈로그**: 사내 에이전트를 검색·발견.
- 간단한 **에이전트 등록** 화면 제공(이름·설명·**URL** 입력).
- 등록된 에이전트를 포털 내 **iframe으로 즉시 실행**.
- **단순 HTML 페이지** 기반의 가벼운 구현.

### 3.2 비목표 (이번 범위 아님)
- 권한·접근 제어(RBAC), SSO 연동, 감사 로그.
- 사용량·비용 청구(Chargeback) 및 모니터링 대시보드.
- 자체 파운데이션 모델(LLM) 학습/호스팅.
- 포털에서의 **직접 LLM 호출/중계**(실행은 외부 에이전트 URL이 담당).
- 외부 고객 대상 공개 마켓플레이스.

| **에이전트 제작자** | 에이전트를 만드는 개발자/현업 | URL만으로 빠르고 간단한 등록 |

- 제작자로서, 이름·설명·**URL**만 입력해 에이전트를 **간단히 등록**하고 싶다.

## 7. 비기능 요구사항 (Non-Functional Requirements)

- **단순성**: 설치 없이 브라우저만으로 사용, 단순 HTML 페이지·직관적인 UI.
- **성능**: 카탈로그 검색 응답 < 1s.
- **반응형**: 데스크톱·모바일 브라우저 지원, iframe 영역도 화면 크기에 맞춰 조정.
- **보안/임베드**: 신뢰 가능한 **사내 도메인 URL** 위주로 등록, iframe `sandbox` 속성 검토.
- **유지보수성**: 단순한 구조로 빠른 기능 추가/수정.

---

## 8. 시스템 아키텍처 개요 (High-Level Architecture)

- **프론트엔드**: 단순 HTML 페이지 — 카탈로그/등록/실행 화면. 선택한 에이전트의 **URL을 iframe `src`로 로드**.
- **백엔드(경량/선택)**: 에이전트 메타데이터 CRUD만 담당. **LLM 직접 호출 없음**.
- **저장소**: 에이전트 메타데이터(이름·설명·URL) 저장용 **정적 JSON 또는 경량 DB**.
- **에이전트 실행**: 각 에이전트는 **독립 URL**로 동작하며, 포털은 이를 **임베드만** 한다.

## 14. 구현 반영 사항 (2026-06-19)

아래 항목은 현재 코드 기준으로 PRD 요구사항에 반영된 구현 현황이다.

### 14.1 현재 구현 범위

- 포털 상단 메뉴를 `HOME`, `AGENT`, `신규등록` 3개로 구성.
- 메인 카탈로그에 빠른 카테고리 버튼(문서 요약/코드 리뷰/데이터 분석/마케팅 카피/고객 응대) 제공.
- `AGENT` 화면을 좌측 에이전트 리스트 + 우측 iframe 실행 영역으로 구성.
- 좌측 리스트는 스크롤 및 검색(`Agent 검색`) 지원.
- 카탈로그 카드에 카테고리 기반 이모지 썸네일을 표시.

### 14.2 데이터/콘텐츠 반영

- 카테고리별 시드 에이전트를 총 25개(카테고리당 5개)로 확장.
- 검색어/태그 필터의 동시 적용으로 결과가 0개가 되는 UX를 보완.
        - 태그 선택 시 검색어 초기화
        - 빠른 카테고리 선택 시 태그 필터 해제

### 14.3 프론트엔드 구조 반영

- 인라인 스타일/스크립트를 제거하고 정적 자산을 외부 파일로 분리.
        - 스타일: `css/main.css`
        - 스크립트 진입점: `js/controller/agentController.js`
- 기존 단일 파일 방식에서 MVC 패턴으로 리팩터링.
        - Model: 데이터/검증/필터링
        - View: DOM 렌더링/폼 처리
        - Controller: 이벤트/화면 전환/실행 흐름

### 14.4 운영/실행 참고

- 모듈 스크립트 정상 로딩을 위해 `file://` 직접 열기 대신 로컬 HTTP 서버 기반 실행 권장.
- 사용량·비용 청구(Chargeback) 및 모니터링 대시보드.
- 자체 파운데이션 모델(LLM) 학습/호스팅.
- 외부 고객 대상 공개 마켓플레이스.

## 8. 시스템 아키텍처 개요 (High-Level Architecture)

```
[사용자 브라우저]
        |
        v
[웹 프론트엔드 (SPA)]
        |  REST API
        v
[백엔드 API 서버] -- [에이전트 메타데이터 DB]
        |
        v
[LLM API (모델 공급자)]
```

- **프론트엔드**: 단일 페이지 웹 앱(SPA) — 카탈로그/등록/실행 화면.
- **백엔드**: 가벼운 REST API 서버 — 에이전트 CRUD, LLM 호출 중계.
- **저장소**: 에이전트 메타데이터(이름·설명·프롬프트) 저장용 경량 DB.
